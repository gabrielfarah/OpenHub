Welcome to OpenHub.

There is currently a vast array of open source projects available on the web, and although they are searchable by name or description in the search engines, there is no way to search for projects by how well they perform on a given set of quality attributes (e.g. usability or maintainability). With OpenHub, we present a scalable and extensible architecture for the static and runtime analysis of open source repositories, initially written in Python.

Millions of repositories were analyzed using this architecture and Github platform. For these repositories, software quality metrics are calculated and stored in a mongodb setup (Check a base example here).

Why it matters?

Using the results from the analyzing process, researchers can perform comparative analysis over the quality metrics and developers can create cool stuff like firefox plugins to extend the github search results with the quality metrics.
